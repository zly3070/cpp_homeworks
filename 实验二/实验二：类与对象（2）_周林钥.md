*（1）程序代码*

```cpp
#include <bits/stdc++.h>

class DataInfo{

		DataInfo();  //无参数构造函数
```

**任务二：const 成员（第 6、7、8 题）**

*（1）第 6 题*

将 `stud` 声明为常对象后，常对象不能调用非常成员函数，因此 `change()` 和 `display()` 均无法调用，编译出错。

*（2）第 7 题*

修改方案：
- 将 `num` 和 `score` 声明为 `mutable`，允许在常成员函数中修改
- 将 `change()` 和 `display()` 声明为 `const` 成员函数

修改后代码编译通过，运行结果为：
```
101 78.5
101 80.5
```

*（3）第 8 题*

将函数参数改为引用传递，避免对象复制，同时可直接修改原对象：
```cpp
void fun(Student& tmp){
    tmp.display();
    tmp.change(101, 80.5);
    tmp.display();
}
```
运行结果：
```
101 78.5
101 80.5
```

*（4）结果分析*

通过 `const` 关键字可以限制成员函数的修改能力，`mutable` 提供了一种在常成员函数中修改特定成员的途径。引用传递避免了复制开销，且可直接修改原对象。

**任务三：静态成员（第 9 题）**

*（1）程序代码*

```cpp
#include <bits/stdc++.h>
using namespace std;

class Sale{
	private:
		int num;
		int quantity;
		double price;
		
		static double discount;
		static double sum;   //总销售额 
		static int n;        //总销售件数 
	
	public:
		Sale(int n, int q, double p) : num(n), quantity(q), price(p) {} 
		
		static void setDiscount(){
			cout << "请输入discount：" << endl;
			cin >> discount;
		}
		
		void calculate(){
			double salePrice = price;
			if (quantity > 10){
				salePrice *= 0.98;
			}
			salePrice *= discount;
			double saleMoney = salePrice * quantity;
			sum += saleMoney;
			n += quantity;
		}
		
		static double average(){
			if (n == 0) return 0;
			return sum / n;
		}
		
		static void display(){
			cout << fixed << setprecision(2);
			cout << "sum:" << sum << endl;
			cout << "average:" << average() << endl;
		}
};

//类外初始化静态数据成员 
double Sale::discount = 0.95;
double Sale::sum = 0;
int Sale::n = 0; 

int main(){
	Sale sales[3] = {
		{101, 5, 23.5},
		{102, 12, 24.56},
		{103, 100, 21.5}
	};
	Sale::setDiscount();
	for (int i = 0; i < 3; i++){
		sales[i].calculate();
	}
	Sale::display();
	return 0;
}
```

*（2）运行结果*

```
请输入discount：
0.95
sum:2624.57
average:21.65
```

*（3）结果分析*

静态成员 `discount`、`sum`、`n` 属于类本身，所有对象共享。静态成员函数 `setDiscount()`、`average()`、`display()` 可以直接通过类名调用，用于操作全局的销售统计数据。类外初始化是静态成员分配内存的必要步骤。

**任务四：友元函数（第 10 题）**

*（1）程序代码*

```cpp
#include <bits/stdc++.h>
using namespace std;

class Date;

class Time{
	public:
		Time(int h, int m, int s) : hour(h), minute(m), sec(s){}
		friend void display(Date&, Time&);
	private:
		int hour;
		int minute;
		int sec;
};

class Date{
	public:
		Date(int y, int m, int d) : year(y), month(m), day(d){}
		friend void display(Date&, Time&);
	private:
		int year;
		int month;
		int day;
};

void display(Date& D, Time& T){
	cout << D.year << "年" << D.month << "月" << D.day << "日" << endl;
	cout << T.hour << ":" << T.minute << ":" << T.sec << endl;
}

int main(){
	Time t1{10, 36, 56};
	Date d1{12, 25, 2024};
	display(d1, t1);
	return 0;
}
```

*（2）运行结果*

```
2024年12月25日
10:36:56
```

*（3）结果分析*

通过友元函数 `display`，可以访问 `Time` 和 `Date` 类的私有成员。友元函数不是类的成员函数，但享有访问私有成员的权限，适用于需要跨类操作且不宜将数据公开的场景。

**四、小结与收获**

通过本次实验，我对 C++ 类的核心机制有了更深入的理解：

- **构造函数与析构函数**：掌握了无参、有参、复制构造函数的定义时机与调用顺序，析构函数在对象销毁时自动执行，用于资源释放。
- **const 成员**：理解了常成员函数不能修改普通成员，但可通过 `mutable` 修饰允许特定成员被修改；引用传递避免了对象复制，提高了效率。
- **静态成员**：静态数据成员属于类本身，所有对象共享，必须在类外初始化；静态成员函数可直接通过类名调用，用于操作全局数据。
- **友元函数**：友元函数可以访问类的私有成员，适用于需要跨类访问的场景，但破坏了封装性，应谨慎使用。
